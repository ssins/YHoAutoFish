# 异环自动钓鱼 - UI设计深度迭代计划

## 1. 现状分析 (Current State Analysis)
- 虽然之前完成了基础的架构重构和图鉴功能，但 UI 的视觉表现还比较基础，没有完全体现出截图中那种现代游戏特有的“赛博朋克+扁平化”的高级质感。
- 截图的 UI 特点分析：
  - **深色主题**：全局以深灰/黑色为主，带有半透明或细微纹理感。
  - **色彩点缀**：以 Logo 的青蓝色（`#6cc9dc`）作为高亮主题色（如光晕、选中边框、进度条）。
  - **卡片设计**：
    - 图鉴/鱼类卡片：鱼的图标放置在一个**青蓝色的圆形背景**中，右下角带有深色胶囊状的**数量角标 (Badge)**。卡片底部往往有不同颜色的区分。
    - 物品卡片（如鱼饵）：采用圆角矩形，上半部分浅色/深色，下半部分带有特定颜色的底栏。
  - **按钮设计**：重要操作按钮（如“开始钓鱼”）采用醒目的**纯白底色+深色文字**，并配有大圆角；次要或关闭按钮采用深色圆底+浅色图标。

## 2. 目标与设计重构 (Proposed Changes)

为了让程序 UI 真正达到现代游戏辅助工具的标准，我们将对 `app.py`, `encyclopedia.py`, `fishing_record.py` 进行深度视觉升级。

### 2.1 全局框架与主控调整 (`app.py`)
- **侧边栏 (Sidebar)**：
  - 颜色加深（如 `#151515`），让主内容区更突出。
  - 导航按钮在选中时，左侧增加一条 `#6cc9dc` 的高亮指示条，背景微微泛亮。
- **主控按钮重塑**：
  - “启动”按钮：改为截图中“开始钓鱼”的风格——**白色背景，深色加粗文字**，大圆角，极具点击欲。
  - “停止”按钮：采用深色底，红色边框和文字，或者红色半透明底。

### 2.2 图鉴记录界面深度复刻 (`encyclopedia.py`)
- **鱼类卡片重构** (核心学习截图1的特点)：
  - 卡片整体背景采用深灰色（`#222222`），边框增加极细的浅色描边。
  - **圆形背景**：利用 `CTkFrame` (宽高一致，圆角=宽/2) 创建一个 `#6cc9dc` (或其他稀有度颜色) 的圆形底板，将鱼类图片放置其中。
  - **数量角标**：在圆形底板的右下角，使用绝对定位 (`.place`) 叠加一个黑底白字的胶囊状 Label（如 `x5`），完美复刻截图中的数量显示。
  - **布局调整**：鱼名放在圆形右侧，或者卡片下方居中。如果是未解锁状态，整个卡片包括圆形背景全部灰度化。

### 2.3 钓鱼记录与统计界面优化 (`fishing_record.py`)
- **数据看板卡片**：
  - 将顶部的统计卡片优化为现代仪表盘风格。卡片内部增加细微的层级（例如标题较暗，数字采用 `#6cc9dc` 且字号加大）。
- **历史记录列表**：
  - 每一行记录采用类似“出没鱼类”的卡片横向排列，或者保留列表形式但增加斑马纹背景和青色悬浮效果。
  - 左侧加上该鱼种的微缩圆形图标。

### 2.4 高级设置与日志界面优化
- 对齐整体的暗黑+青蓝点缀风格。所有的 `CTkSwitch` 和 `CTkSlider` 的激活颜色严格统一为 `#6cc9dc`。
- 将各种参数设置分组放入带有深色背景和圆角的 `CTkFrame` 容器中，模拟截图2中“选择钓竿”那种模块化的分区感。

## 3. 假设与依赖 (Assumptions & Decisions)
- **视觉限制**：`customtkinter` 不支持复杂的阴影 (Drop Shadow) 和复杂的渐变色 (Gradient)。我们将通过颜色对比（深灰背景 vs 纯黑/亮灰容器）、精细的边框（Border Width=1）和圆角（Corner Radius=8~15）来模拟高级感。
- **动态图标**：继续使用 `PIL` 处理未解锁鱼类的灰度效果。角标使用 `.place(relx=..., rely=...)` 实现悬浮层叠效果。

## 4. 验证步骤 (Verification Steps)
1. 运行程序，观察侧边栏导航是否有明确的选中指示，且“启动”按钮变为醒目的白色大按钮。
2. 进入“图鉴记录”，确认鱼类图片是否嵌套在圆形背景中，且右下角有类似 `x0` 的黑色胶囊角标。
3. 确认未获取的鱼，其卡片整体（包括圆形背景）是否呈现灰色，且开关“查看全部鱼”后逻辑依然正常。
4. 检查各分区的层级感，确保没有违和的纯色大色块，整体色调和谐统一（深黑、深灰、青蓝）。
