# 异环自动钓鱼 - UI全面升级与图鉴功能开发计划

## 1. 现状分析 (Current State Analysis)
- 当前应用使用 `customtkinter` 的 `CTkTabview` 布局，分为“主控面板”和“高级设置”。
- 没有持久化的钓鱼历史记录功能。
- 自动钓鱼成功时（`_handle_result`），仅通过 `self.fish_count += 1` 累加数字，未识别具体鱼种和重量。
- 缺少 OCR 库来提取游戏内的中文文本。

## 2. 目标与设计重构 (Proposed Changes)
基于用户的截图风格（深色半透明背景、圆角卡片、特定按钮高亮），将主色调定为 `#6cc9dc`（从 `logo.jpg` 提取）。抛弃原有的 `CTkTabview`，改用更现代的 **侧边栏导航 + 主视图切换** 架构。默认主页改为新增的“钓鱼记录”。

### 2.1 UI 架构与导航重构
- **侧边栏 (Sidebar)**：
  - 顶部左上角放置 `logo.jpg`（统一色调 `#6cc9dc`）。
  - 添加导航按钮：`[钓鱼记录]`, `[图鉴记录]`, `[运行日志]`, `[高级设置]`。
  - 底部保留“启动/停止”控制区和状态指示灯。
- **运行日志 (Run Log)**：原“主控面板”重命名并移入独立的 Frame。
- **高级设置 (Advanced Settings)**：保留原有功能，并增加 OCR 识别区域的微调选项（鱼名 ROI 和 重量 ROI）。

### 2.2 钓鱼记录面板 (Fishing Record)
- **核心统计区 (Top)**：总钓获数、总运行时长、成功率、已解锁鱼种进度。
- **图表分析区 (Middle)**：使用 `matplotlib` (`FigureCanvasTkAgg`) 绘制饼图或柱状图，展示各稀有度鱼类的占比分布。
- **最近捕获列表 (Bottom)**：可滚动的历史记录列表，显示：时间、鱼种名称、重量、稀有度。

### 2.3 图鉴记录面板 (Encyclopedia Record)
- **网格展示区**：读取 `异环鱼类图鉴资源`，按稀有度（废品、绿色、蓝色、紫色、金色）分组或整体展示。
- **灰度化处理**：未捕获的鱼，使用 `PIL.ImageOps.grayscale` 处理为灰色；已捕获的显示原色，并标注最高重量和捕获次数。
- **查看全部鱼 (Toggle)**：开启后，隐藏捕获数量统计，所有鱼类图片强制恢复正常彩色，供纯展示用。

### 2.4 核心逻辑更新 (状态机与 OCR)
- **新增 OCR 模块**：引入 `cnocr`（轻量级中英文识别库，适合纯 Python 环境）或 `easyocr`，用于在钓鱼成功界面提取文本。
- **识别逻辑 (`state_machine.py`)**：
  - 触发成功条件后，截取屏幕指定区域：
    - 鱼名 ROI：`X: 40%-64%`, `Y: 15.5%-27%`。
    - 重量 ROI：`X: 40%-60%`, `Y: 63%-74%`。
  - 调用 OCR 提取文本（去除重量文本中的非数字和"g"）。
  - 将数据结构化（时间、鱼名、重量、稀有度）并保存。
- **数据持久化 (`record_manager.py`)**：新增类负责读写 `records.json`，管理历史记录和图鉴解锁状态。

## 3. 假设与依赖 (Assumptions & Decisions)
- **依赖库增加**：需要安装 `cnocr` (或 `easyocr`), `matplotlib`, `pillow`。
- **图鉴匹配**：通过 OCR 识别出的鱼名，直接与 `异环鱼类图鉴资源` 文件夹下的图片文件名（如 `锦鲤.png`）进行精确或模糊匹配。
- **UI 尺寸**：为容纳图表和图鉴网格，主窗口默认尺寸将从 `650x550` 扩大至 `1000x700`。

## 4. 验证步骤 (Verification Steps)
1. 启动应用，检查默认界面是否为“钓鱼记录”，且左侧侧边栏、Logo 及主色调 `#6cc9dc` 正确应用。
2. 切换至“图鉴记录”，验证未捕获鱼类是否为灰色，点击“查看全部鱼”是否恢复彩色。
3. 开启自动钓鱼（或使用测试图片模拟），验证 OCR 是否能准确提取指定 ROI 的鱼名和重量。
4. 验证成功捕获后，“钓鱼记录”的图表和列表是否实时更新，并且“图鉴记录”中对应的鱼被点亮。
5. 重启应用，验证 `records.json` 数据持久化正常。
