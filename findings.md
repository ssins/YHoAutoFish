# Findings & Decisions

## Requirements
- 使用中文输出。
- 结合用户提供的四张截图，整理“异环”自动钓鱼程序的完整 Python 实现方案。
- 本轮不写代码，只给出完整代码编写思路、模块设计、算法方案与实施步骤。
- 钓鱼流程已知：
- 初始界面：右下角存在钓鱼按钮提示，按 `F` 进入钓鱼。
- 等待阶段：角色抛竿后等待鱼上钩。
- 上钩提示阶段：出现“鱼上钩了，快点击按钮上鱼！”时，再按 `F` 进入正式拉鱼阶段。
- 拉鱼阶段：界面上方存在移动的绿色耐力条，以及需要通过 `A`/`D` 控制的黄色竖条。
- 控制目标：尽量让黄色竖条保持在绿色条内，从而降低左侧鱼耐力；同时右侧鱼线会随时间下降，降到 0 则失败。

## Research Findings
- 当前项目目录只有 `main.py` 一个 Python 文件，尚未形成实际业务结构。
- `main.py` 是 PyCharm 默认模板，说明项目基本处于空白起步状态。
- 该任务天然适合分成“视觉识别 + 状态机 + 输入控制 + 标定配置 + 日志调试”五大部分。
- 由于用户已经提供关键 UI 截图，可以优先采用 ROI + 模板匹配/颜色阈值/简单 OCR 的混合方案，而不是一开始就上复杂模型。

## Technical Decisions
| Decision | Rationale |
|----------|-----------|
| 采用有限状态机驱动流程 | 游戏交互是明显的阶段式流程，状态机更稳、更易调试 |
| 采用 ROI 定位而非全屏盲扫 | UI 位置相对稳定，局部识别更快、更抗误判 |
| 拉鱼阶段优先使用像素/颜色条分析 | 顶部绿色条和黄色竖条属于典型 HUD 元素，做颜色与几何检测成本最低 |
| 将控制策略设计为“短按 A/D + 死区 + 冷却” | 比持续按住更容易抑制抖动和过冲 |
| 保留 OCR 作为提示文字识别的备选方案 | “鱼上钩了”提示可用模板，也可用 OCR 兜底 |
