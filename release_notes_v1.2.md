# YHo AutoFish v1.2 正式发布

v1.2 是一次面向稳定性、兼容性和用户体验的正式修复版本。相比 v1.1，本版本重点解决高分辨率适配、树林/白天环境溜鱼识别、失败后不继续、捕获记录上限、结算误判、停止后仍发送按键、用户接管保护和在线热更新等问题。

> 本程序仅用于图像识别、桌面自动化流程学习与个人技术研究。请勿用于商业牟利、代练代刷、批量传播或任何侵权用途。使用自动化工具可能违反游戏或平台规则，风险由使用者自行承担。

## 重要变化

- 从 v1.2 起，程序会强制请求管理员权限。拒绝管理员权限时，程序不会继续运行。
- 发布包内置独立更新器 `YHoUpdater.exe`，支持后续从程序内一键热更新。
- 捕获记录不再限制为 1000 条，阶段总结不会再因为历史记录长度固定而误判“上次总结过了”。
- 溜鱼识别已从高成本模板思路回到性能更稳定的颜色识别，并对 HUD 区域、绿条颜色和黄色游标做了严格限制。

## 相比 v1.1 的完整修复与优化

### 1. 分辨率、高 DPI 和全屏适配

- 增加 Windows DPI awareness，统一 Win32 客户区坐标和截图坐标。
- 修复 2K/4K/全屏/高 DPI 下模板匹配区域与真实画面错位的问题。
- 模板缩放从固定范围改为按客户区高度动态适配，提升 1080p、2K、4K 和不同窗口比例下的识别稳定性。
- 等待抛竿、等待上钩、结果判定等阶段增加超时和恢复流程，避免误识别后永久等待。

### 2. 右下角钓鱼入口识别

- 识别入口从单张 `F` 图标扩展为多模板库，支持已有多张 F 图标资源。
- 增加边缘匹配、前景 mask、多阈值二值匹配和分辨率缩放。
- 加入 `Q/E/R` 右侧组合控件作为钓鱼初始界面的辅助判定，减少半透明 UI 受背景颜色影响导致的误判。
- 增加“开始钓鱼按钮”和准备界面识别逻辑，避免停在准备界面时无法恢复。

### 3. 失败、空杆和恢复流程

- 修复鱼儿溜走后不重新钓鱼的问题。
- 失败后记录一次空杆/失败尝试，释放按键并等待回到可抛钩界面。
- 修复 F 按键未被游戏接收时会一直等待上钩的问题，增加快速重试和恢复流程。
- 修复等待阶段错误把“等待上钩时的 F 图标”当作初始抛竿图标的问题。
- 修复失败界面误按 ESC 导致回到钓鱼准备界面的情况。
- 增加统一的本轮状态清理，避免失败后的 PID、按键、计时器状态残留影响下一轮。

### 4. 成功结算识别

- 成功结算不再只依赖单一文字特征，新增组合判定：
  - 点击关闭提示
  - 获得钓鱼经验
  - 重量单位 `g`
  - 结算画面局部特征
- 修复失败后被误判为成功的问题。
- 优化结算检测节奏，减少成功上鱼后等待过久才开始 OCR 的情况。
- 结算关闭后更快检测回到可抛钩状态，减少下一轮等待。
- 停止指令触发后，结算和待机流程会检查停止状态，避免停止后继续发送 `F`。

### 5. 溜鱼耐力条识别

- 针对树林、深绿背景、水面、白天强光环境重构识别逻辑。
- 绿色耐力条识别优先使用用户提供的真实耐力条颜色范围，减少背景绿色参与计算。
- 限制识别到 HUD 水平带内，避免整块 ROI 中背景树木、草地或水面绿色把中心点拉偏。
- 对黄色游标增加颜色、形状、位置和与绿条关系的筛选，减少识别到其他黄色发光背景。
- 对 HUD 突然变宽、调试视图异常拉长等情况增加区域约束，尽量只识别真正的溜鱼条区域。
- 减少高成本模板回放和过重匹配，降低白天画面卡顿导致的游标静止问题。
- 优化 `mss` 截图异常后的容错，降低截图失败对实时控制的影响。

### 6. 溜鱼控制算法和默认参数

- 调整默认跟鱼力度，使 A/D 官方移动速度降低后仍能更积极追踪耐力条。
- 默认死区更窄，游标轻微偏离时也会更快响应。
- 优化前馈、中心安全区、反向切换、按键保持与释放逻辑。
- 增加更细的高级设置分类，便于按“溜鱼控制、流程与超时、识别与判定、安全接管”调整。
- 高级设置保存逻辑更准确：用户改动后又改回已保存值时，不再显示未保存警示，也不会启用保存按钮。

### 7. 捕获记录和阶段总结

- 移除捕获记录 1000 条硬裁剪，完整历史不再丢失。
- 每条记录新增单调递增 `record_id`，根节点新增 `next_record_id`。
- 阶段总结游标从 `last_history_len` 改为 `last_record_id`，历史记录变多后仍能准确统计新增记录。
- 旧版 `records.json` 会自动迁移，补齐缺失的 `record_id`，并尽量把旧总结游标转换为新游标。
- 保留 GUI 查询缓存，避免记录变多后全量刷新造成卡顿。

### 8. 用户接管保护

- 新增用户接管检测：用户点击游戏窗口或在游戏内按键时，程序会释放按键并暂停。
- 不检测单纯鼠标移动，避免过于激进。
- 点击程序左上角悬浮窗不会触发接管暂停。
- 暂停提示弹窗显示在游戏窗口中央，醒目告知用户程序已暂停。

### 9. UI 和交互优化

- 悬浮窗支持收起成长横条，减少遮挡游戏视野。
- 优化运行日志和悬浮窗日志滚动条风格。
- 新增关于按钮和关于界面，展示版本、作者、项目地址、使用范围和风险提示。
- 在标题栏展示版本号和检查更新入口。
- 高级设置改为左侧分类、右侧参数区滚动，底部保存栏固定且更醒目。
- 新增用户协议和反侵权协议入口。

### 10. 在线热更新

- 新增 GitHub Release 热更新流程：
  - 自动检查更新
  - 版本比较
  - 下载 `latest.json`
  - 下载 zip 更新包
  - SHA256 校验
  - 启动独立更新器
  - 覆盖程序文件并重启
- `latest.json` 支持 UTF-8 BOM 容错。
- 下载地址改用 GitHub 官方 `releases/latest/download/<asset>` 形式，避免 `v1.3` / `1.3` tag 写法差异导致下载失败。
- 自动检查改为启动后后台检查一次，并在运行期间默认每 30 分钟轮询静态 `latest.json`；手动点击版本入口会立即重新检查，不受自动轮询间隔影响，发现新版后会停止后台轮询。
- 自动检查失败不再频繁刷运行日志，降低对用户的干扰。
- 更新前增加用户协议和反侵权协议二次确认。
- 独立更新器保护用户数据，不覆盖 `config.json`、`records.json`、`records.db`、`logs/`、`screenshots/`、`captures/` 等。
- 更新器增加自身进程保护，避免误把更新器自身当作主程序等待或覆盖。
- 更新下载包、解压目录和更新器运行副本改为写入程序目录 `.updates/`，不再默认占用 C 盘 `%TEMP%`。

### 11. 发布与打包

- 主程序和更新器均要求管理员权限。
- 发布脚本会自动同步版本号到 `version_info.txt`。
- 发布脚本会生成：
  - `YHoAutoFish-v1.2-windows.zip`
  - `latest.json`
- 发布脚本支持 `-NotesFile` 或 `-Notes`，可把本说明写入 `latest.json`，让程序更新弹窗显示详细更新日志。
- 发布包会移除作者本地测试 `records.json`，避免污染用户数据。

## 升级建议

- v1.1 用户建议升级到 v1.2。
- 升级前可备份旧目录中的 `records.json` 和 `config.json`。
- 解压新版后，优先使用新版默认参数进行几轮测试，再按需要调整高级设置。
- 如果你在特殊钓点仍遇到识别异常，请开启调试溜鱼视图并反馈截图。

## 已知说明

- 国内访问 GitHub 可能不稳定，自动更新失败时可以稍后手动检查或手动下载 Release 包。
- 游戏 UI 被遮挡、极低帧率、非标准画面比例或 UI 缩放异常，仍可能影响识别。
- OCR 识别鱼名和重量依赖结算画面清晰度，偶发识别错误时会在日志中提示。
